# HA9004 - Format multiline raw string literal indentation

## Description

Report multiline raw string literals whose closing delimiter is not indented with the opening delimiter.

For raw string literals returned directly from a method, HA9004 also reports when the opening delimiter starts on the same line as `return`, or when the opening and closing delimiters are not indented one level deeper than the `return` statement.

## Motivation

Multiline raw strings are easier to read when their delimiter indentation matches the surrounding code. Aligned delimiters keep generated code, JSON, SQL, and other embedded text blocks predictable while preserving the runtime string value.

## How to fix violations

Indent the raw string content and closing delimiter to match the opening delimiter. For direct return statements, move the opening delimiter to the line after `return` and indent the literal one level deeper.

## Code with Diagnostic

```csharp
public static string Create() {
	return """
{
	"value": true
}
""";
}
```

## Code with Fix

```csharp
public static string Create() {
	return
		"""
		{
			"value": true
		}
		""";
}
```
