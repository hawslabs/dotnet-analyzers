# HA9005 - Keep 1TBS brace settings consistent

Sources: [`OneTrueBraceStyleAnalyzer`](../../packages/analyzers/src/OneTrueBraceStyleAnalyzer.cs)

| Analyzer          | `OneTrueBraceStyleAnalyzer` |

## Description

Report C# files whose effective `.editorconfig` sets `indent_brace_style` to `1TBS` or `OTBS` while a related C# formatting option disagrees with the expected one true brace style value.

HA9005 reports one diagnostic for each conflicting option. `csharp_prefer_braces` must be `true`; when the analyzer can observe an inline or compiler-provided `IDE0011` severity, that severity must be `error`.

## Motivation

`indent_brace_style` is a general EditorConfig setting, but the .NET formatter uses C#-specific formatting options for brace and newline behavior. HA9005 keeps those settings aligned.

## How to fix violations

When `indent_brace_style` is `1TBS` or `OTBS`, set the matching C# formatting options in `.editorconfig`:

```editorconfig
csharp_prefer_braces = true
dotnet_diagnostic.IDE0011.severity = error

csharp_new_line_before_open_brace = none
csharp_new_line_before_else = false
csharp_new_line_before_catch = false
csharp_new_line_before_finally = false
csharp_new_line_before_members_in_object_initializers = false
csharp_new_line_before_members_in_anonymous_types = false
csharp_new_line_between_query_expression_clauses = false
```

## Code with Diagnostic

```editorconfig
indent_brace_style = 1TBS
csharp_prefer_braces = false
csharp_new_line_before_open_brace = all
```

## Code with Fix

HA9005 does not provide a code fix. Update the conflicting `.editorconfig` entries manually to the values listed above.
