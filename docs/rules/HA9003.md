# HA9003 - Keep short First calls with their receiver

## Description

Report short chained `First*` invocations whose receiver and invocation can fit on one line while the single multiline argument remains partly on that line.

## Motivation

Keeping a short `First*` invocation with its receiver makes the chain easier to read, while moving the multiline argument to following lines keeps the predicate body visually balanced.

## How to fix violations

Move the `.First*(` call up to the receiver line and place the multiline argument body on following lines.

## Code with Diagnostic

```csharp
var value = items
	.FirstOrDefault(item => string.Equals(
		item.Name,
		name,
		StringComparison.Ordinal
	)!;
```

## Code with Fix

```csharp
var value = items.FirstOrDefault(
	item => string.Equals(
		item.Name,
		name,
		StringComparison.Ordinal
	)
)!;
```
