# HA9002 - Keep parameter-list continuations with the closing parenthesis

Sources: [`HangingListClosingParenAnalyzer`](../../packages/analyzers/src/HangingListClosingParenAnalyzer.cs), [`HangingListClosingParenCodeFixProvider`](../../packages/analyzers/src/HangingListClosingParenCodeFixProvider.cs)

## Description

Report parameter lists whose continuation token is stranded on the line after the closing parenthesis. This includes expression-bodied member `=>` tokens and primary-constructor base-list `:` tokens.

## Motivation

Keeping `) =>` and `) :` together makes the declaration boundary clear and avoids a dangling token that visually looks detached from the signature.

## How to fix violations

Move the continuation token to the closing parenthesis line. If the parameter list also needs expansion, put each parameter on its own line first.

## Code with Diagnostic

```csharp
private static Task Create(
	string name
)
	=> Task.CompletedTask;
```

## Code with Fix

```csharp
private static Task Create(
	string name
) => Task.CompletedTask;
```
