# HA0001 - Put hanging-list closing parenthesis on its own line

Sources: [`HangingListClosingParenAnalyzer`](../../packages/analyzers/HangingListClosingParenAnalyzer.cs), [`HangingListClosingParenCodeFixProvider`](../../packages/analyzers/HangingListClosingParenCodeFixProvider.cs)

| Enabled           | True                                     |

| Code fix provider | `HangingListClosingParenCodeFixProvider` |

## Description

Report hanging multiline argument lists and parameter lists whose closing parenthesis is not placed on its own line and aligned with the line that contains the opening parenthesis.

## Motivation

Hanging multiline lists are easier to scan when their closing parenthesis is visually paired with the opening line. This keeps nested calls and declarations predictable, reduces visual drift, and makes formatting intent clearer during reviews.

## How to fix violations

If the first argument or parameter starts on the line after the opening parenthesis, move the closing parenthesis to its own line and indent it to the same level as the line that contains the opening parenthesis.

When the closing parenthesis is attached to a multiline raw string literal delimiter, the code fix also reindents the raw string delimiter and content so the source is formatted while preserving the string value.

## Code with Diagnostic

### Argument list

```csharp
CallTarget(
	1,
	2);
```

### Parameter list

```csharp
private static void Test(
	int first,
	int second) {
}
```

### Raw string argument wrapper

```csharp
VerifyAnalyzerAsync(InMethodBody(
	"""
CallTarget(
	1,
	2);
"""));
```
