# HA0001 - Put hanging-list closing parenthesis on its own line

Sources: [`HangingListClosingParenAnalyzer`](../../packages/analyzers/src/HangingListClosingParenAnalyzer.cs), [`HangingListClosingParenCodeFixProvider`](../../packages/analyzers/src/HangingListClosingParenCodeFixProvider.cs)

| Topic             | Value                                    |
| :---------------- | :--------------------------------------- |
| Id                | `HA0001`                                 |
| Severity          | Warning                                  |
| Enabled           | True                                     |
| Category          | Formatting                               |
| Code fix          | Yes                                      |
| Analyzer          | `HangingListClosingParenAnalyzer`        |
| Code fix provider | `HangingListClosingParenCodeFixProvider` |

## Description

Report hanging multiline argument lists and parameter lists whose closing parenthesis is not placed on its own line and aligned with the line that contains the opening parenthesis. HA0001 also reports split argument and parameter lists when one item remains beside the opening parenthesis while another item starts on a later line.

For expression-bodied members, HA0001 also reports a hanging parameter list when the closing parenthesis is already aligned but the `=>` token is stranded on the next line. For primary constructors with a base list, HA0001 reports parameter lists that stay on the opening line while the base-list colon is stranded on the next line.

For argument lists, HA0001 also reports multiline raw string literal arguments whose closing delimiter is not indented with the raw string opening delimiter when the surrounding hanging list is otherwise already aligned.

## Motivation

Hanging multiline lists are easier to scan when their closing parenthesis is visually paired with the opening line. This keeps nested calls and declarations predictable, reduces visual drift, and makes formatting intent clearer during reviews.

## How to fix violations

If any argument or parameter in a list starts on a later line, put every argument or parameter on its own line. Put the closing parenthesis on its own line and indent it to the same level as the line that contains the opening parenthesis.

For expression-bodied members, keep the closing parenthesis and `=>` token together on the same line.

For primary constructors with a base list, keep the closing parenthesis and base-list colon together on the same line.

When a hanging argument list wraps a multiline raw string literal, the code fix also reindents the raw string delimiter and content so the source is formatted while preserving the string value.
